## Midterm Examination Reflection

1. What is Git used for?
Git, based from my experience and understanding, is a system used to track changes in files and manage project history over time. It is truly beneficial to different individual like a student like me that is currently takng BSIT and has a goal to become a successful developer someday because it is use to collaborate on the same project without overwriting each other's work.

2. What is the difference between Git and GitHub?
Based on what I learned, Git is a version control tool that we install on our own computer, and it is used to track the changes we make in our project files. GitHub on the other hand is a website where we can upload our Git repositories online, so we can share our code with groupmates, work together on the same project, and also have a backup in case something happens to our local files.

3. Why is branching useful?
Branching is useful because it lets me work on new features or bug fixes without messing up the main code. If I make a mistake in my branch, the main branch is still safe and working. I can test my changes first, and only merge them back to main once I am sure that everything works properly. This way, the project stays stable even while I am still experimenting or making changes.

4. What command did you use to upload your local commits to GitHub?
I just used `git push` to upload my local commits to the GitHub repository.
